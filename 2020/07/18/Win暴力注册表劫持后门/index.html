    sethc注册表劫持

    注：此方法很老套，并且动静很大这里只作为一个记录与思路拓展

    进程名称： sethc.exe

    描述：Windows的粘滞键。是按5下shift后，windows就执行了system32下的sethc.exe。

    它本来是为不方便按组合 键的人设计的。

    位置：c:\windows\system32\sethc.exe

    信息来自：百度百科

    regini.exe是Windows NT框架中一个实用命令行工具，可以通过命令行或脚本进行创建、修改、删除注册表项值、注册表项权限。

    将以下内容保存为regeini.ini，然后使用命令regini regini.ini进行修改注册表

    说明：

    1、当系统或用户运行sethc.exe时，实际是运行注册表指定的程序

    2、并为这个表项设置权限为Administrators用户组和System为只读权限

    HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Image File Execution Options\sethc.exe[2 19]

    顺带一提

    dllcache大约有350MB，2000左右个文件，大部分是动态链接库（.dll ）和可执行文件（.exe ），是个系统隐藏文件夹。 dllcache作为备份是在使用“系统文件检测”时发挥作用的，它实际上“准备”了你可能会用到的所有重要系统文件，是一个备用的系统文件库。

    要说dllcache就不得不说下windows的文件保护机制。很多诸如动态链接库（.dll ）和可执行文件（.exe ）之类的文件对于系统的稳定运作是非常重要的，如果这些文件被不小心删了或替换了，就有可能造成系统的不稳定，甚至系统崩溃。虽然我们一般不会有意去删除或替换这些文件，但实际上装软件的时候，特别是一些版本老的软件的时候，覆盖一些共享的系统文件是常有的事。

    针对这一点，windows就有一个文件保护的后台服务。默认情况下，该服务一直处于启用状态，监视着所有受保护的系统文件，如果发现替换或移动受保护的系统文件企图，它能直接阻止。当然windows并不阻止所有这样的企图，它允许有windows数字签名文件替换现有文件，这样你的系统才可以更新和升级。

    %SystemRoot%\system32\Dllcache\

    也就是说，当我们替换了system32目录下的文件时，系统会从备份把我们恶意放入的sethc.exe替换成原版，这个需要注意。
