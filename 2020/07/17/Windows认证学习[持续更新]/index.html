      1. 0x01 NTML（NT LAN Manager） HASH与NTLM
      2. 0x02 LM Hash

    学习自倾旋的彻底理解Windows认证的议题：https://payloads.online/

    以下文章是根据文章及视频的学习记录

    0x00 Windows本地认证

    在windows登录时，操作系统会使用用户输入的账户密码与存储在本地的密码(SAM文件)去做对比。

    %SystemRoot%\system32\config\sam

    需要注意的是，为了保证系统的安全性，Windows是不保存明文密码的，只保留密码的Hash。

    Hash，一般翻译做散列、杂凑，或音译为哈希，是把任意长度的输入（又叫做预映射pre-image）通过散列算法变换成固定长度的输出，该输出就是散列值。这种转换是一种压缩映射，也就是，散列值的空间通常远小于输入的空间，不同的输入可能会散列成相同的输出，所以不可能从散列值来确定唯一的输入值。简单的说就是一种将任意长度的消息压缩到某一固定长度的消息摘要的函数。 —-百度百科

    注：如果被攻击者拿到SAM文件，即可破解系统密码

    0x01 NTML（NT LAN Manager） HASH与NTLM

    早期SMB协议在网络上传输明文口令。后来出现 LAN Manager Challenge/Response验证机制，简称LM，它是如此简单以至很容易就被破解。微软提出了WindowsNT挑战/响应验证机制，称之为NTLM。已经有了更新的NTLM v2以及Kerberos验证体系。NTLM是windows早期安全协议，因向后兼容性而保留下来。NTLM是NT LAN Manager的缩写，即NT LAN管理器。 —-百度百科

    NTLM Hash是由数字与字母组成的32位的值并且是经过hex、unicode、md4进行三层加密所得。

    上一节说到，Windows系统本身是不存储明文密码的，所以在SAM文件里存储的是密码的Hash值，在我们登录系统所输入密码并按下登录后，系统会把我们输入的密码转换为Hash值，并与SAM里面的Hash值做对比。

    其中，NTLM与NTLM Hash之间的关系为：

    0x02 LM Hash

    LM Hash（基本淘汰）是NTLM协议的前身，它们两个虽然认证机制相同，但是它们的算法不同。

    可知LM协议认证过程中需要LM Hash作为根本凭证进行参与认证。下面大致看下流程

    将所有小写字母转换为大写字母
>123ABC // 未达到7个字符
• 将密码转化为16进制，分两组，填充为14个字符,空余位使用0x00字符填补
• >31323341424300000000000000
• 将密码分割为两组7个字节的块
• >31323341424300 00000000000000 // 16进制
• 将每组转化为比特流，不足56Bit则在左边加0
• >31323341424300 ->(转换为二进制) 110001001100100011001101000001010000100100001100000000-> (补 足56Bit) 00110001001100100011001101000001010000100100001100000000
• 将比特流按照7比特一组，分出8组，末尾加0

由于后者都为0，结果可想而知，那就都是0;
• 将每组比特流转换为16进制作为被加密的值，使用DES加密，字符串 “KGS!@#$%”为Key(0x4B47532140232425)，得到8个结果 ，每个 结果转换为16进制。
• -> 00110000100110001000110001101000000101000001001000001100 00000000
• ->30988C6814120C00 -> DES(30988C6814120C00) -> 48-D7-EB-91- 2F-5E-69-7C
• 由于我们的密码不超过7字节，所以后面的一半是固定的:
• AA-D3-B4-35-B5-14-04-EE
• 连接两个DES加密字符串。这是LM哈希。
• 48-D7-EB-91-2F-5E-69-7C-AA-D3-B4-35-B5-14-04-EE

    此时可以看出这个算法的弱点在于DES算法的Key（KGS!@#$%）是固定的，如果攻击者拿到Key，就能解出明文。而且攻击者可以根据LM Hash的特性，能够判断用户密码是否≥7位。
